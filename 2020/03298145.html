什么是拓扑排序 Topological Sorting

来源：https://songlee24.github.io/2015/05/07/topological-sorting/

在图论中，拓扑排序（Topological Sorting）是一个有向无环图（DAG, Directed Acyclic Graph）的所有顶点的线性序列。且该序列必须满足下面两个条件：

  • 每个顶点出现且只出现一次。
  • 若存在一条从顶点 A 到顶点 B 的路径，那么在序列中顶点 A 出现在顶点 B 的前面。

算法描述：

  • 统计每个点的入度
  • 将每个入度为 0 的点放入队列（Queue）中作为起始节点
  • 不断从队列中拿出一个点，去掉这个点的所有连边（指向其他点的边），其他点的相应的入度 - 1
  • 一旦发现新的入度为 0 的点，丢回队列中

有向无环图（DAG）才有拓扑排序，非DAG图没有拓扑排序一说。

名词解释：topological sorting - 拓扑排序，topological order - 拓扑序

BFS经典模板之拓扑排序， 很难很多细节，这个答案我自己写的，不确定对不对，但能过所有的LintCode 和LeetCode case。。。因为我总觉得这个解法没有考虑到连通性。。。可能 len(edges) != n - 1 已经考虑了连通性了，如果小于这个是不连通的，大于这个是有环的。

证明：可以用反证法来解释在 len(edges) == n -1 的情况下，只需要验证有没有环就能解释有没有树的存在。Case 1: 如果图是连通的，那么 环存在等价于树不存在； Case 2：如果图是不连通的，那么必然有一个连通分支存在环，否则的话 len(edges) != n - 1。可以想象下有k >= 1棵树的情况。即不连通，不存在环同时有len(edges) == n -1 的情况根本不存在。

思路：用拓扑排序来判断图上是否有环和孤立点

嗯，九章推荐用union find来做。

Union Find 标准模板未经优化版

Union Find 经典模板之优化版

优化有两点：1. Find 函数中找到根结点后，把所有相关联的结点的root 都update 下，比如之前是 1->2, 2->3，现在变为 1->3, 2->3。2. Union 函数中将结点数量小的那队加到数量多的那个根结点上，这样可以提高下次Find的速度。这两个优化具体在时间复杂度上很难体现出来，因为它们是相辅相成的，需要大量数据才能看出与未经优化版的差异。

思路：1. 比标准模板多了 weights 这个类变量，主要用于记录当前结点与其父结点的距离，比如：weights[a] 的值等于 a / parents[a]； 2. find 函数用了dfs，没有用普通的while循环来写。因为要update每个结点的weights值，只能通过stack的思想，由浅入深递归的去update；3. union函数update 父结点的weight值，这个是最难理解的。已知 x, y 和 x/y = val 的值，难点在于我们要update的既不是 x 也不是 y 的 weight值，我们想要做的是 update parents[x] 或者 parents[y] 的值，这个要取决于取谁做儿子。以下用图形来说明，如果 parents[x] = y，那么weights[p[X]]的值到底是多少？w[p[X]]的值用w[x], w[y]以及 val 来表示。其实很简单，画个图就能把思路理清。
