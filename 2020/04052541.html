(4)Tree based DFS 学习笔记

Apr 5 2020 Notes an hour read (About 7396 words)

从最简单的DFS开始：二叉树上的深度优先搜索。

一些知识点：

  • DFS 一般用来找path比较好，BFS用来找node比较好。
  • DFS 出口一般是 root == None 的情况，有时也需要考虑叶子结点的情况。
  • DFS ：traversal 通常没有返回值（回溯？？？），divide and conquer一定要有返回值，否则无法知道处理后的结果。
  • 大部分题目都推荐用 divide and conquer 来做，因为 traversal使用了全局变量，很容易出错。
  • 时间复杂度$O(N)$，所有的结点访问一次且仅一次。
  • LintCode 902 和 95： 如何用iteration的方式来实现中序遍历。page 21 of PPT。阅读全文并背诵。
  • BST 是某种形式的有序集。
  • 先序遍历，中序遍历，后序遍历，这里面的先、中、后都是相对访问根的顺序来说的。简单的说，先序遍历最先访问root，中序遍历在中间访问root，后序遍历最后才访问root。
  • Divide and conquer: divide() and combine()

DFS 小总结

  • 如果是要求出所有的路径，那么通常要用backtracking的技巧，本质原因是python在传数组的时候传递的是reference，在递归的时候已经改变了path，要回溯到上层的时候必须要pop出来才能变回本来的状态。
  • 如果只是要求出路径的深度，那么没必要backtracking。可以把当前结点的深度+1传递到子树中，无论子树中怎么递归都不会改变当前结点的深度。
  • 以上两种情况都需要在遇到叶子结点时进行结算。

一些练习

  1. LeetCode 1120. Maximum Average Subtree

    divide and conquer 标准模板

    思路：分治法最难的就是返回值的确定。这里面返回的是个三元数组：1、以当前结点为根的subtree中结点的数目，2、以当前结点为根的subtree的平均值，3、目前为止平均值最大的那个数。

    stopping condition 应该选择不会影响上层结果的数，这里面由于所有的数都是正的故 if not root: return (0, 0, 0) 是可行的，否则可以选择 if not root: return (0, 0, -float(“inf”))。因为比的是最大值。

    divide and conquer 标准模板

    traversal 标准模板， 涉及 backtracking

    注意：traversal 是先update 路径，再DFS，这也刚好与 divide and conquer 相反。

  10. LeetCode 230. Kth Smallest Element in a BST

    divide and conquer 标准模板，极难，参考了网上思路后，想了很久才搞出来

    思路：秉乘着不定义全局变量，没有额外传入参数的原则来写的。返回值有两个：1、子树结点的个数，2、找到的子树。需要注意的是如果2即子树找到的话，那么1的值就不重要了，这也是为什么在div_conq中最后返回的是 return (left_size + 1 + right_size, left_subtree or right_subtree)， 这是为了整体显得整洁美观。

    思路：python中没有引用，返回值也不是数组，貌似只能定义全局变量来保存遍历结果？？？引用额外传入参数好像也不行。。。

    中序遍历的迭代实现方式

    思路：一路向左append，然后pop的过程中，如果有右儿子就append右儿子，继续一路向左append，以此类推，这样就能保证栈顶始终是最小的元素。

  17. LeetCode 450. Delete Node in a BST

    二叉树的删除，很难，对照着LeetCode解答才写出来

    核心思路：1、如果target是叶子，那么直接删除；2、如果target有右子树，那么找到其右子树中最小的点，复制其值到target后，再用递归的手段将那个点删除；3、在1和2都不成立的前提下，那么target必然有左子树，于是先找到左子树中值最大点，复制其值到target，再用递归的手段将那个点删除。

    上述算法的迭代实现

    思路：与递归一模一样，用迭代写的话，有很多细节需要考虑。不背下来的话，基本不可能在面试中写出来。

  21. LeetCode 105, 106

    从中序先序 / 中序后序 遍历数组中构造二叉树，这种题技巧性太强了。第一眼看到了都没啥想法。递归就那么几行代码，非递归的解法试图去弄明白，最终还是放弃了。

    例如：106. Construct Binary Tree from Inorder and Postorder Traversal

    思路：后序遍历顺序： left -> right -> root。以后序遍历的最后一个结点为根结点root，然后分别构造左右子树。首先在中序数组中将root的序号i找到，这个i告诉了我们左子树的长度是i - 1，右子树的长度是 n - i。对应于后序数组中我们就有以下结果：左子树 <-> [0 : i]， 右子树 <-> [i, -1]。

  22. LeetCode 108. Convert Sorted Array to Binary Search Tree

    从有序数组中构造二叉搜索树，技巧性也非常的强

    思路：每次取数组的中位数做root，这样左侧的数肯定比root小，右侧的数肯定比root 大。

小总结：感觉21 和 22本质上都是一种divide and conquer的方法。有返回值，左右儿子的构造过程都是在递归中完成的。相当于先把left 和 right构造完成，再把它们link到root的左右儿子上面去。但这种题的难点在于，在递归构造左右儿子的过程中，递归方程的参数并不是简单的变成root.left和root.right，而是要根据题意做相应的调整。

  1. LeetCode 979.Distribute Coins in Binary Tree

    dfs遍历，极难，思路不对的话根本做不出来

    思路：dfs返回的是当前子树的不平衡数，即，与子树结点数目的差，多为正，少为负。当前子树的move数不仅与其左右子树的不平衡数有关，还与其所有的子树的不平衡数相关，故不能用divide and conquer的方法，只能能过定义全局变量的方法来update这个move数。

  4. LeetCode 1110. Delete Nodes And Return Forest

    DFS 遍历和divide and conquer的结合体。太难，根本想不到，对照着评论区大神的解法才写出来

    思路：看似是divide and conquer，实则是遍历。因为在divide之前会做与当前结点相关的运算。然后代入到下层子树中去。一个node只有没被删除才有可能成为一个子树的根结点。除此之外，要想成为根结点，它的你结点必须被删除了。结合这两点不难理解算法。DFS返回的实际上是当前结点是否被删除的状态：是的话返回None，否的话就直接返回此结点。最后在递归出栈的时候改变结点的left和right 值，如果某个儿子被删除了，则它的父结点就会指向None。
